Présentation de l'application

Python Timer est une application web développée pour suivre et enregistrer avec précision le temps passé sur des tickets ServiceNow. Conçue avec Flask, cette solution professionnelle permet aux équipes IT et aux équipes de support de mesurer leur productivité et d'optimiser leur gestion du temps grâce à un système de chronométrage en temps réel.

L'application se distingue par son utilisation avancée des WebSockets pour offrir une expérience utilisateur interactive où les mises à jour sont visibles instantanément pour tous les clients connectés. Grâce à l'intégration avec l'API ServiceNow, Python Timer permet une synchronisation fluide entre le suivi du temps et la gestion des tickets, offrant ainsi une solution complète pour les professionnels de l'IT.

Technologies utilisées

Backend

  • Flask (v2.3.3) : Framework web léger en Python qui sert de base à l'application
  • Flask-SocketIO (v5.3.4) : Extension permettant la communication en temps réel entre client et serveur via WebSocket
  • Python-dotenv (v1.0.0) : Gestion des variables d'environnement à partir d'un fichier .env
  • Requests (v2.31.0) : Bibliothèque pour effectuer des requêtes HTTP vers des API externes
  • PyTZ (v2023.3) : Gestion des fuseaux horaires et des dates
  • Gevent (v23.9.1) et Gevent-WebSocket (v0.10.1) : Serveur WSGI asynchrone optimisé pour les applications WebSocket

Intégration

  • ServiceNow API : Intégration avec la plateforme ServiceNow pour récupérer et mettre à jour des tickets

Interface utilisateur

  • HTML/CSS/JavaScript : Interface utilisateur web rendue par Flask
  • WebSockets : Communication bidirectionnelle en temps réel entre le navigateur et le serveur

Fonctionnalités principales

Gestion des tickets ServiceNow

  • Récupération et affichage des tickets depuis ServiceNow
  • Consultation des détails d'un ticket spécifique
  • Vérification de la connexion avec ServiceNow

Système de chronométrage (Timer)

  • Démarrer un timer sur un ticket spécifique
  • Mettre en pause le timer d'un ticket
  • Arrêter le timer et enregistrer le temps passé
  • Mise à jour en temps réel de l'état des timers pour tous les clients connectés
  • Stockage temporaire des timers actifs en mémoire

Dashboard et visualisations

  • Page d'accueil avec tableau de bord
  • Vue "sonar" pour une visualisation graphique des données

Gestion des utilisateurs

  • Formulaires de connexion et d'inscription
  • Authentification des utilisateurs

Configuration et paramétrage

  • Gestion des paramètres de l'application
  • Configuration des identifiants ServiceNow

Mises à jour en temps réel

  • Communication en temps réel via WebSockets pour mettre à jour les timers
  • Thread en arrière-plan pour les mises à jour périodiques des timers

Architecture de l'application

Python Timer est construit selon une architecture moderne qui sépare clairement le frontend du backend, tout en permettant une communication en temps réel grâce aux WebSockets. Le backend Flask gère l'API, l'authentification et les intégrations externes, tandis que le frontend offre une interface réactive qui s'actualise instantanément sans nécessiter de rafraîchissement de la page.

L'application utilise un serveur WSGI asynchrone (Gevent) pour gérer efficacement de multiples connexions simultanées et assurer des performances optimales, même sous charge. Le stockage en mémoire des timers actifs permet d'obtenir des réponses très rapides, tandis que l'intégration avec ServiceNow garantit que toutes les données sont correctement synchronisées avec le système de gestion des tickets.
